# Calendar Booking TypeSpec

Контракт описывает API для сервиса бронирования с двумя контекстами:

- `Admin` — владелец календаря по умолчанию, без регистрации/логина.
- `Public` — гость, который бронирует слот без аккаунта.

## Доменные сущности

- `Owner` — заранее заданный владелец календаря.
- `EventType` — тип встречи с названием, описанием и длительностью.
- `Slot` — временной интервал для выбранного типа события.
- `Booking` — факт записи гостя на слот.

## Правила контракта

- Слоты выдаются в окне ближайших 14 дней (если `from`/`to` не заданы).
- Нельзя создать две записи на один интервал времени даже для разных типов событий.
- При конфликте занятости API возвращает `409 SlotConflictError`.

## Публичный сценарий гостя

1. `GET /public/event-types`
2. `GET /public/event-types/{eventTypeId}/slots`
3. `POST /public/bookings`
